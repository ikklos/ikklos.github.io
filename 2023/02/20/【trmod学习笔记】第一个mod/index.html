【trmod学习笔记】第一个mod

第一个模组问世

首先让我们通过tml自带的创建模组功能，创建一个新模组

在tml的 创意工坊 -> 开发模组 -> 创建模组 一栏创建即可，可以设置模组名称、展示名称、模组作者、模板剑等等，需要注意的是模组名称和展示名称是不一样的，模组名称是指你的源代码文件按中这个模组的名字，是不允许带空格等非法符号的。这样我们第一个模组就建成了，当然没有这么简单，这才刚刚开始，接下来我们认识一下生成的这些文件

我的模组里面都有什么

通过点击“打开源码文件夹”按钮就可以直接跳转到Modsources目录，建议记住这个路径，之后再找模组源码的时候就可以不用打开tml了。

目前能看到的是一些必要的文件，也就是一些最基础的东西，我们来了解一下这些文件是做什么的。

  1. [模组名称].cs - Mod类 就在这里了，这是整个模组的中心。如果不理解类的概念没关系，先迷迷糊糊地跳过就好。后缀.cs是C#代码文件的后缀。 每一个模组只能有一个Mod类 ，这是肯定的。起初它可能很简单，但慢慢地我们可以在它上面实现更多的功能。
  2. description.txt - 里面包含着这个模组的介绍，可以在游戏里显示，用来编辑我们模组的介绍信息。
  3. build.txt - 包含一些版本信息、作者名字、展示名称之类的内容。
  4. icon.png - 一个80 X 80的图片，是在游戏里显示的我们模组的图标，另外我们还可以新建一个icon_workshop.png 作为steam创意工坊上的模组图标，最大可以到 512 X 512。
  5. [模组名称]. csproj - 这是模组的项目文件，vs支持打开这种文件格式，用来组织项目，非常方便。
  6. Properties/launchSettings.json - 与上一个文件相联系，它包含了一些debug的路径信息，最好别删
  7. Items/[物品名称].cs - 在Items文件夹下，它是物品的源码文件
  8. Items/[物品名称].cs - 与物品源码在同一个目录下，注意要和物品名字匹配，而且 记得要用英文 。

生成模组

目前这个模组还没法玩，因为它不是.tmod文件，怎么办呢，我们需要使用tml的生成功能，也就是在刚刚的地方点击“生成并重新加载” ，这样我们的模组就真正可以游玩了。它帮助我们把我们的源码文件打包成一个.tmod文件，如果我们需要给朋友玩玩我们的模组，只需要把这个文件给他就行了。

修改基础属性

这可以说是最没有技术含量的操作了，相信大伙在游玩其他游戏的时候（尤其是单机游戏）都有过修改代码的经历，其实当时修改的代码就多数是数值，但是那个跟现在的情况还不一样，我们当初只是修改了配置文件里面的数值信息，我们碰不到源码，现在我们可以直接在源码里面改了， 这或许也是一种进步 。

我们游玩刚刚建成的模组，发现有个模板剑可以用，但它的属性太低了不够爽，怎么改呢？我们直接打开这个物品的源码文件，上文提到过，就是Item文件夹下的.cs文件，用文本编辑器打开它，或者直接用你的IDE打开整个模组的项目文件，在IDE中找到这个源码文件。无论通过哪种方式，我们看到了物品的源码。

在里面找到Item.damage = 50; 这一行，随便改改，另外再随便改改 Tooltip.SetDefault("This is a basic modded sword."); 这一行，注意C#的语法要求，然后再 重新点击“生成并重新加载”生成模组 ，进游戏看看效果。不出意外的话我们就能看到新的伤害数值和新的提示文本了。

至于其他的属性，我们可以自由地修改一下，再 重新生成 ，进游戏看看效果，这可以帮助我们更好地理解这些属性的作用，而且可以通过这种简单的操作激发我们的兴趣。

引用

如果我们想要使用自定义的武器来发射自定义的弹幕或者实现其他类似的功能，我们需要 引用其他的类 ，注意这与我们创建新的物品时 继承 的操作并不相同，我们需要借用 Modcontent.SomethingType 方法，其中 something 就是你想要引用的东西类型，可以是“Item”、“Projectile”、“NPC”等等。如果我们想让武器射出模组弹幕，只需要在 ModItem.Setdefaults 方法中写 Item.shoot = ModContent.ProjectileType<弹幕的名字>() 即可。C#中的“方法”一词与“函数”的概念类似，只不过不同程序设计语言中叫法不一样罢了，我们需要习惯这种叫法，是为了方便C#中的面向对象思想。另外，如果我们只是想要射出原版的弹幕，我们只需要使用ProjectID即可，不需要再使用其他方法来引用弹幕类。

Hook（钩子）的使用

钩子是什么

根据度娘的解释，hook（钩子）是windows提供的一种消息处理机制平台，是指在程序正常运行中接受信息之前预先启动的函数，用来检查和修改传给该程序的信息，（钩子）实际上是一个处理消息的程序段，通过系统调用，把它挂入系统。每当特定的消息发出， 在没有到达目的窗口前，钩子程序就先捕获该消息，亦即钩子函数先得到控制权。这时钩子函数即可以加工处理（改变）该消息，也可以不作处理而继续传递该消息，还可以强制结束消息的传递(return)。 百度的解释太不友好了，简而言之， 钩子起到消息处理的作用。

为了进一步理解钩子函数的作用，我们研究一下Windows的 消息 机制。Windows系统以消息处理作为它的控制机制，无论是系统还是应用都产生消息，移动鼠标，单击某个键位，或者是应用带给系统的变化等等都会产生一系列消息，应用和系统通过消息响应，通过消息来与各个窗口通信。

每个窗口背后都有一个这样的程序，称作窗口程序，它是一段程序脚本，可以进行消息和事件的处理等等，Windows中为每条消息制定了一个消息编号，例如WM_ACTIVATE消息的意思就是“活动窗口”，也就是正在操作的这一个窗口。还有其他的消息例如Load，MouseDown等等。命令按钮也有类似的消息处理程序，它处理的消息诸如 WM_LBUTTONDOWN和
WM_RBUTTONDOWN，即MouseDown事件。

钩子函数很强大，但在强大的功能背后也有弊端，所以合理使用钩子是很重要的。

tml的“钩子”

tml中的“钩子”是我们身为模组制作者可以使用的一个功能，可以使用的钩子取决于我们正在使用的类，例如，Mod类下有一个Load钩子，描述如下 Override this method to add most of your content to your mod. Here you will call other methods such as AddItem. This is guaranteed to be called after all content has been autoloaded. 钩子函数定义为virtual 类型，意味着它具有基础实现，在使用时，我们需要 override 它，也就是重写。即使这样，原先的 基础实现也不会消失 。

tml中的钩子是模组执行代码的基本因素，每一个钩子都会在游戏的更新循环中，在合适的时机被调用。

举个例子，ModItem.Shoot 就是一个钩子，当每次武器需要发射弹幕的时候这个钩子就会被调用，制作者通过 重写 添加的一些代码就会在这时被执行。

Index 和 Type

这两个都是整数类型，但如果混淆它们则有可能在编译中出现bug，接下来我们看看它们的区别。

Type可以看作是“静态的”，泰拉瑞亚中每一个物品都有一个Type，而且是独一无二的，用来区分不同的物品（这里的物品包括很多，包括弹幕之类）

Index 是另一种概念，游戏在运行时会开辟一部分空间，用这个临时数组来存储 活动的 物品，也就是说它是动态的，每一件物品并不会像 Type 那样有一个固定的编号。比如说 Main.projectile[] 里面存储了目前世界中所有的弹幕，每一个弹幕可以通过 whoAmI 字段获取它们自己的编号。

创建自己的模组类

想要创建自己的模组类，或更具体地说，想要创建自己的模组物品，使用的最简单的方式就是 派生类 或者叫做 继承 ，比如MyItemClass : ModItem 这样就创建了一个物品，冒号后面加ModItem表示它继承了ModItem，同理我们也可以继承其他的类，但只能同时继承一个类，也就是说我们不能弄一个既是物品又是弹幕的类。我们做了继承ModItem的操作之后，ModItem中的钩子就可以使用了。
